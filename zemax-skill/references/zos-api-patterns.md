# ZOS-API Patterns

## Connection Triage

Prefer Standalone Application for automated runs that create, open, optimize, analyze, or save files without needing the visible GUI.

Use Interactive Extension only when the user wants Python to operate on the currently visible OpticStudio session. The independent Interactive Extension dialog must be open, and the instance number must match the command.

Minimum success criteria:

- `ZOSAPI_NetHelper.dll` resolves from the selected OpticStudio root.
- `CreateNewApplication()` or `ConnectAsExtension(instance)` returns an application object.
- `IsValidLicenseForAPI` is true when available.
- `PrimarySystem` is not null before reading or modifying optical data.

## Python.NET Loading

Use `pythonnet`/`clr` with the OpticStudio installation DLLs. Add references to:

- `ZOSAPI_NetHelper`
- `ZOSAPI_Interfaces`
- `ZOSAPI`

When enum assignment is unstable, parse enums explicitly:

```python
from System import Enum

operand.ChangeType(Enum.Parse(ZOSAPI.Editors.MFE.MeritOperandType, "MTFA"))
```

Avoid assigning raw strings or integers to enum-typed API calls unless a local probe confirms it works.

## Merit Function Editor

Use `GetOperandCell(MeritColumn.<name>)` rather than raw numeric cell indexes. This is more robust across operands and OpticStudio versions.

For `MTFA` and `MTFT`, the useful cell layout observed in practice is:

| Meaning | MeritColumn | Value type |
|---|---|---|
| Field | `Param3` | `IntegerValue` |
| Frequency | `Param4` | `DoubleValue` |
| Wave | `Param6` | `IntegerValue` |

Always verify operand cell meanings by exporting or inspecting the MFE after a new operand family is introduced.

## Analysis Export

For automated validation, export text results for FFT MTF, Standard Spot, Ray Fan, Wavefront, and Distortion when direct data APIs are not convenient. Parse structured values from those files only after preserving the raw text alongside the summary metrics.

Recommended run artifacts:

- `requirements-applied.json`
- `design-log.jsonl`
- `stage-<name>.json`
- `metrics-<stage>.json`
- `analyses/<stage>/mtf.txt`
- `analyses/<stage>/distortion.txt`
- `analyses/<stage>/spot.txt`

## Common Failure Modes

- Interactive Extension appears connected but has no valid API license or `PrimarySystem`; use Standalone for batch design.
- Generic autogenerated merit functions may optimize focus while ignoring magnification, track length, distortion, or real MTF frequency.
- A lower merit value can hide field imbalance or distortion regression.
- Localized OpticStudio installations may return object display names in a non-English UI; prefer enum names where possible.
- Some API object methods are discoverable only by probing; keep small introspection scripts and JSON dumps for fragile NSC work.
